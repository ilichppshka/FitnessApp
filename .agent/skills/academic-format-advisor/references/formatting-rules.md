# Правила оформления учебного документа по эталонному DOCX

Источник правил: правила ниже извлечены из эталонного файла `Пример оформления. Диплом-Патурин-Илья-0901.docx`. Не пересказывать и не использовать содержание диплома; использовать только параметры оформления, структуру и повторяющиеся паттерны.

> Важно: это не официальный текст ГОСТ и не замена методическим указаниям вуза. Если пользователь предоставляет методичку, задание кафедры или требования преподавателя, они имеют приоритет над этими правилами. Если требования противоречат эталону, явно указать конфликт и спросить, какой источник считать главным.

## Назначение

Использовать эти правила, когда пользователь спрашивает, как правильно оформить диплом, ВКР, курсовую, реферат, отчет, отчет по практике или другой учебный документ. Этот файл предназначен для скилла-консультанта: он должен объяснять правила, проверять соответствие и формулировать рекомендации. Он не должен сам редактировать `.docx`; для реального изменения файла надо передавать технические инструкции отдельному docx-скиллу.

## Общие параметры страницы

| Параметр | Правило |
|---|---|
| Формат страницы | A4 |
| Ориентация | Книжная |
| Верхнее поле | 2 см |
| Нижнее поле | 2 см |
| Левое поле | 3 см |
| Правое поле | 1 см |
| Отступ верхнего колонтитула | 1,25 см |
| Отступ нижнего колонтитула | 1,25 см |
| Переплет | 0 см |
| Колонки | Одна колонка |

## Нумерация страниц и колонтитулы

- Номер страницы размещается в верхнем колонтитуле по центру.
- Используется автоматическое поле номера страницы `PAGE`.
- Первая страница считается титульной и не должна показывать номер страницы.
- Со второй страницы номер отображается в верхнем колонтитуле; в эталоне страница содержания отображается как страница 2.
- Нижний колонтитул в эталоне не используется.
- Номер страницы наследует общий стиль документа: Times New Roman, 14 pt, черный цвет, без жирного начертания.

## Основной текст

| Параметр | Правило |
|---|---|

| Размер | 14 pt |
| Цвет | Черный |
| Выравнивание | По ширине |
| Межстрочный интервал | 1,5 |
| Интервал перед абзацем | 0 pt |
| Интервал после абзаца | 0 pt |
| Абзацный отступ первой строки | 1,25 см |
| Переносы и разрывы | Не оставлять одиночные заголовки внизу страницы; крупные разделы начинать с новой страницы |

Рекомендации:

- Не имитировать абзацный отступ пробелами или табуляцией; использовать настройку первой строки абзаца.
- Не добавлять пустые абзацы между обычными абзацами основного текста.
- В обычном тексте не использовать произвольные шрифты, цвета, подчеркивания и декоративные выделения.
- Термины, названия программ, организаций и технологий оформлять обычным шрифтом, если иное не требует методичка.

## Структура документа

В эталоне используется такая крупная структура:

1. Титульный лист.
2. Содержание.
3. Введение.
4. Основные разделы с нумерацией.
5. Заключение.
6. Список литературы.
7. Приложения.

Правила разрывов:

- После титульного листа ставится разрыв страницы.
- После содержания ставится разрыв страницы.
- `Введение` начинается с новой страницы.
- Каждый крупный раздел первого уровня начинается с новой страницы.
- `Заключение` начинается с новой страницы.
- `Список литературы` начинается с новой страницы.
- Каждое приложение начинается с новой страницы.

## Титульный лист

В эталонном файле титульный лист представлен только как заглушка `Титульник`, поэтому точные правила титульного листа из этого файла не извлекаются.

Что рекомендовать пользователю:

- Для титульного листа запросить методичку вуза, шаблон кафедры или пример утвержденного титульника.
- Не выдумывать точную структуру титульного листа, если пользователь не предоставил образец.
- Сохранять общий принцип: титульный лист является первой страницей документа, но номер на нем не отображается.
- После титульного листа обязательно ставить разрыв страницы.

## Содержание

| Параметр | Правило |
|---|---|
| Заголовок | `Содержание` |
| Выравнивание заголовка | По центру |
| Абзацный отступ заголовка | 0 см |
| Интервал после заголовка | 24 pt |
| Шрифт | Times New Roman, 14 pt |
| Межстрочный интервал | 1,5 |
| Тип содержания | Автоматическое оглавление Word |
| Уровни в содержании | 1–2 |
| Лидер табуляции | Точки |
| Номер страницы | Справа |
| Позиция правой табуляции | Около 16,98 см от левого края текстовой области |

Правила:

- Содержание должно включать заголовки 1-го и 2-го уровней.
- Подразделы 3-го и 4-го уровней в содержание не включать, если пользователь не просит иначе.
- В содержании используются точки-лидеры между названием раздела и номером страницы.
- Номера страниц выравниваются по правому краю.
- После содержания начинается новая страница.
- При изменении структуры документа содержание должно быть обновлено.

## Заголовки

### Общие правила заголовков

- Шрифт заголовков: Times New Roman, 14 pt, черный.
- Заголовки не заканчиваются точкой.
- Нумерация разделов пишется вручную или через корректно настроенную многоуровневую нумерацию, но должна визуально соответствовать эталону.
- Между номером и названием раздела ставится пробел.
- Заголовок не должен отрываться от следующего текста; использовать `keep with next` / `не отрывать от следующего`.
- Не оставлять заголовок последней строкой страницы.

### Заголовки первого уровня

Используются для крупных разделов и специальных частей документа.

Примеры формата:

```text
Введение
1 Общая часть
2 Специальная, практическая, опытно-экспериментальная часть
3 Экономическая часть
Заключение
Список литературы
Приложение А
```

Правила:

- Каждый заголовок первого уровня начинается с новой страницы.
- Для `Введение`, `Заключение`, `Список литературы` и заголовков приложений используется выравнивание по центру.
- Для нумерованных глав допускается выравнивание по левому краю, как в эталоне.
- Абзацный отступ у центрированных заголовков — 0 см.
- После заголовков первого уровня может быть интервал до 24 pt, но для нумерованных глав в эталоне часто используется 0 pt перед следующим подзаголовком.
- Номер главы пишется без точки после номера: `1 Общая часть`, а не `1. Общая часть`, если ориентироваться на основной текст эталона. В содержании встречается вариант `1. Общая часть`; при создании нового документа лучше придерживаться единого варианта без точки после номера главы.

### Заголовки второго уровня

Примеры:

```text
1.1 Постановка задачи
1.2 Обоснование проектных решений
2.1 Анализ задачи
3.1 Расчет основной и дополнительной заработной платы с отчислениями на социальное страхование
```

Правила:

- Использовать числовую нумерацию вида `1.1`, `1.2`, `2.1`.
- Точка после последней цифры не ставится.
- Заголовок не заканчивается точкой.
- Подраздел второго уровня не обязательно начинать с новой страницы.
- В эталоне стиль второго уровня имеет увеличенные интервалы до/после, но первый подраздел сразу после заголовка главы может иметь интервал перед 0 pt.
- Такие заголовки включаются в содержание.

### Заголовки третьего уровня

Примеры:

```text
1.1.1 Обоснование необходимости разработки
2.3.6 Тестирование
3.1.4 Расчет отчислений на социальное страхование
```

Правила:

- Использовать числовую нумерацию вида `1.1.1`, `2.3.6`, `3.1.4`.
- Точка после последней цифры не ставится.
- Подраздел третьего уровня не начинается с новой страницы, если это не требуется логикой текста.
- В эталоне заголовки третьего уровня не включены в содержание.

### Заголовки четвертого уровня

Примеры:

```text
2.4.3.1 Работа пользователя
2.4.3.2 Работа администратора
2.4.3.3 Техника безопасности при работе на компьютере
```

Правила:

- Использовать числовую нумерацию вида `2.4.3.1`.
- Не включать такие заголовки в содержание, если пользователь не просит расширенное содержание.
- Использовать только при необходимости глубокой структуры; не дробить документ чрезмерно.

## Абзацы и интервалы вокруг структурных элементов

- Основной текст: 0 pt до и 0 pt после.
- Заголовок `Содержание`: после 24 pt.
- Подписи рисунков: в эталоне обычно имеют центрирование и интервал после около 24 pt.
- Подписи таблиц: перед подписью около 24 pt, после 0 pt.
- Не использовать пустые строки вместо настроенных интервалов, кроме случаев явного разрыва страницы или визуального разделения в приложениях.

## Списки

### Маркированные списки

В эталоне маркированные списки оформлены как настоящие списки Word, а не вручную набранные символы.

Правила:

- Шрифт пунктов списка: Times New Roman, 14 pt.
- Межстрочный интервал: 1,5.
- Выравнивание: как основной текст, обычно по ширине.
- Абзацный отступ визуально соответствует общему абзацному отступу 1,25 см.
- Не добавлять пустые строки между пунктами одного списка.
- Если пункты являются продолжением вводной фразы, начинать их со строчной буквы.
- В конце пунктов использовать `;`, а у последнего пункта — `.`.
- Для обычных перечней использовать маркер в виде тире/маркера Word, визуально соответствующий эталону.

Пример логики пунктуации:

```text
Для достижения цели необходимо решить следующие задачи:
– изучение предметной области;
– разработать программный модуль;
– провести тестирование.
```

### Вложенные списки

В эталоне встречается структура, где внешний уровень — маркированный пункт, а внутренний уровень — нумерация с закрывающей скобкой.

Пример:

```text
– инженер по техническому обслуживанию и ремонту:
  1) ежедневное обслуживание;
  2) мониторинг состояния оборудования.
```

Правила:

- Внешний уровень: маркированный список.
- Внутренний уровень: арабские цифры с закрывающей скобкой: `1)`, `2)`, `3)`.
- Внутренний уровень имеет увеличенный отступ; в эталоне первая строка вложенных пунктов визуально смещена примерно до 2,5 см.
- Не смешивать ручные номера и автоматическую нумерацию внутри одного списка.
- Следить, чтобы пунктуация была единообразной: `;` внутри списка, `.` в последнем пункте.

### Нумерованные списки

- Для списка литературы используется нумерация `1.`, `2.`, `3.`.
- Для вложенных перечислений внутри текста используется нумерация `1)`, `2)`, `3)`.
- Для заголовков используется иерархическая нумерация без точки в конце: `1`, `1.1`, `1.1.1`.

## Таблицы

### Подписи таблиц

Подпись таблицы размещается перед таблицей.

Формат:

```text
Таблица N – Название таблицы
```

Для таблиц внутри разделов используется сквозная нумерация по документу: `Таблица 1`, `Таблица 2`, `Таблица 3` и т.д.

Правила подписи:

- Выравнивание: по левому краю.
- Шрифт: Times New Roman, 14 pt.
- Межстрочный интервал: 1,5.
- Интервал перед подписью: около 24 pt.
- Интервал после подписи: 0 pt.
- Между номером таблицы и названием используется тире `–`.
- Точка в конце подписи не ставится.
- Если нужно указать единицы измерения, их можно размещать справа на той же строке с помощью табуляции, как в эталоне.

Пример:

```text
Таблица 2 – Основная заработная плата работников        в рублях
```

### Оформление самой таблицы

| Параметр | Правило |
|---|---|
| Стиль таблицы | Table Grid / сетка таблицы |
| Границы | Одинарные линии всех внешних и внутренних границ |
| Толщина границ | Около 0,5 pt |
| Ширина таблицы | Обычно по ширине текстовой области, около 17 см |
| Шрифт в таблице | Times New Roman, 14 pt |
| Межстрочный интервал в таблице | Обычно одинарный или компактный; не использовать 1,5, если таблица становится громоздкой |
| Абзацный отступ в ячейках | Обычно 0 см |
| Заголовки столбцов | Центрировать |
| Числовые значения | Центрировать |
| Текстовые значения | Допускается выравнивание по левому краю |

Рекомендации:

- Не делать таблицу шире области текста.
- Не использовать декоративную заливку, если ее нет в методичке.
- Не оставлять таблицу без подписи.
- Если таблица переносится на следующую страницу, использовать строку `Окончание таблицы N` над продолжением.
- При продолжении таблицы повторять строку с номерами или заголовками столбцов.

### Продолжение таблицы

Формат:

```text
Окончание таблицы N
```

Правила:

- Размещать перед продолжением таблицы.
- Не создавать новую самостоятельную подпись `Таблица N`; использовать именно `Окончание таблицы N`.
- В продолжении таблицы повторять структуру столбцов.

## Рисунки и изображения

### Размещение рисунков

- Рисунки размещаются по центру страницы.
- Ширина крупных схем и диаграмм может занимать почти всю ширину текстовой области, до примерно 17 см.
- Рисунок не должен выходить за поля страницы.
- Перед рисунком и после подписи оставлять достаточно визуального пространства, но не имитировать интервалы множеством пустых строк.
- Крупные рисунки, которые не помещаются в основной текст, лучше выносить в приложение.

### Подписи рисунков

Подпись рисунка размещается после рисунка.

Формат:

```text
Рисунок N – Название рисунка
```

Правила:

- Выравнивание: по центру.
- Шрифт: Times New Roman, 14 pt.
- Межстрочный интервал: 1,5.
- Интервал после подписи: около 24 pt.
- Между номером рисунка и названием используется тире `–`.
- Точка в конце подписи не ставится.
- Для рисунков в приложениях использовать номер с буквой приложения: `Рисунок А.1`, `Рисунок Б.1`, `Рисунок В.1`.

## Формулы

В эталоне формулы и номера формул оформлены через таблицу без границ: формула находится в левой широкой ячейке, номер формулы — в правой узкой ячейке.

Формат номера:

```text
(3.1)
```

Правила:

- Формулу размещать по центру строки.
- Номер формулы размещать справа в круглых скобках.
- Использовать нумерацию по разделу: `(3.1)`, `(3.2)`, `(3.3)`.
- Таблица для формулы не должна иметь видимых границ.
- Левая часть занимает примерно 15 см, правая — примерно 2,26 см.
- После формулы можно давать расшифровку через строку `где`.
- Не использовать рисунки формул, если формулу можно набрать средствами Word.

## Список литературы

### Заголовок

- `Список литературы` начинается с новой страницы.
- Заголовок выравнивается по центру.
- Абзацный отступ заголовка: 0 см.
- Шрифт: Times New Roman, 14 pt.

### Записи списка литературы

- Используется нумерованный список вида `1.`, `2.`, `3.`.
- Шрифт: Times New Roman, 14 pt.
- Межстрочный интервал: 1,5.
- Абзацный отступ визуально соответствует 1,25 см.
- Записи идут без пустых строк между ними.
- Для электронных источников указывать ссылку в квадратных скобках и дату обращения в круглых скобках, если это требуется методичкой.
- Единый стиль оформления важнее смешения разных библиографических паттернов.

Примеры типов источников из эталона:

- ГОСТ / стандарт.
- Книга.
- Официальная документация или сайт.
- Электронный ресурс с датой обращения.

## Приложения

### Заголовок приложения

Формат:

```text
Приложение А
(обязательное)
```

или:

```text
Приложение Г
(информационное)
```

Правила:

- Каждое приложение начинается с новой страницы.
- Заголовок `Приложение А`, `Приложение Б` и т.д. выравнивается по центру.
- Абзацный отступ заголовка: 0 см.
- После строки `Приложение А` в эталоне используется 0 pt.
- Следующая строка указывает статус приложения: `(обязательное)` или `(информационное)`.
- Статус приложения выравнивается по центру.
- После строки со статусом используется увеличенный интервал, около 24 pt.

### Нумерация объектов в приложениях

- Рисунки в приложениях нумеруются с буквой приложения: `Рисунок А.1`, `Рисунок Б.1`.
- Листинги в приложениях нумеруются с буквой приложения: `Листинг Г.1`.
- Для каждого приложения использовать собственную последовательность нумерации.

## Листинги и код

В эталоне листинг размещен в приложении.

### Подпись листинга

Формат:

```text
Листинг Г.1 – пример кода функции GetHashFile сервиса HashService
```

Правила:

- Подпись листинга размещается перед кодом.
- В приложениях использовать номер с буквой приложения: `Г.1`, `Г.2` и т.д.
- Между номером и названием использовать тире `–`.
- Точка в конце подписи не ставится.

### Код

| Параметр | Правило |
|---|---|
| Шрифт | Courier New |
| Размер | 10 pt |
| Межстрочный интервал | 1,0 |
| Выравнивание | По левому краю |
| Абзацный отступ | 0 см |
| Цвет | Черный |

Рекомендации:

- Не переносить код в основной текст, если он большой; выносить в приложение.
- Сохранять отступы кода.
- Не использовать скриншоты кода вместо текста, если текстовый листинг подходит.

## Сноски, ссылки и перекрестные ссылки

В эталоне основное внимание уделено ссылкам на рисунки, таблицы, приложения и листинги.

Правила:

- При первом упоминании объекта в тексте ссылаться на него явно: `на рисунке 1`, `в таблице 1`, `в приложении А`, `в листинге Г.1`.
- Названия объектов в тексте писать со строчной буквы, если они не начинают предложение: `рисунок`, `таблица`, `приложение`, `листинг`.
- Нумерация объекта в ссылке должна совпадать с подписью.
- Для будущего docx-редактирования лучше использовать поля Word / перекрестные ссылки, но консультирующий скилл должен как минимум проверять логическую согласованность номеров.

## Единицы измерения и числовые данные

- В таблицах единицы измерения можно указывать в подписи справа: `в рублях`, `в часах`.
- Дробные числа в русскоязычном документе оформлять через запятую: `11428,57`.
- Проценты писать с пробелом между числом и знаком `%`, если это требует методичка; в эталоне встречаются значения в таблицах без сложного форматирования.
- Не смешивать разные форматы денежных единиц в одной таблице.

## Ограничения извлеченных правил

- Титульный лист в эталоне заменен заглушкой, поэтому точное оформление титульного листа нужно брать из методички или отдельного шаблона.
- Эталон отражает конкретную работу и может содержать отдельные несогласованности, например различие между формой номера главы в содержании и в тексте. В новых документах лучше делать стиль единообразным.
- Эти правила описывают фактическое оформление эталона, а не полный набор требований ГОСТ.
- Для официального соответствия ГОСТ/вузу всегда сверять список литературы, титульный лист, приложения и оформление иллюстраций с актуальной методичкой.
